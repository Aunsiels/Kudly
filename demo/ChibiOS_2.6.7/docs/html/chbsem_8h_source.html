28 /**
29  * @file chbsem.h
30  * @brief Binary semaphores structures and macros.
31  *
32  * @addtogroup binary_semaphores
33  * @details Binary semaphores related APIs and services.
34  *
35  * <h2>Operation mode</h2>
36  * Binary semaphores are implemented as a set of macros that use the
37  * existing counting semaphores primitives. The difference between
38  * counting and binary semaphores is that the counter of binary
39  * semaphores is not allowed to grow above the value 1. Repeated
40  * signal operation are ignored. A binary semaphore can thus have
41  * only two defined states:
42  * - <b>Taken</b>, when its counter has a value of zero or lower
43  * than zero. A negative number represent the number of threads
44  * queued on the binary semaphore.
45  * - <b>Not taken</b>, when its counter has a value of one.
46  * .
47  * Binary semaphores are different from mutexes because there is no
48  * the concept of ownership, a binary semaphore can be taken by a
49  * thread and signaled by another thread or an interrupt handler,
50  * mutexes can only be taken and released by the same thread. Another
51  * difference is that binary semaphores, unlike mutexes, do not
52  * implement the priority inheritance protocol.<br>
53  * In order to use the binary semaphores APIs the @p CH_USE_SEMAPHORES
54  * option must be enabled in @p chconf.h.
55  * @{
56  */
